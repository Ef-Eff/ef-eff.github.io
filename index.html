Twitter Trends

What is it?

Last project from GA and the first use of Ruby on Rails and an SQL database (PostgreSQL), with the same old AngularJS frontend learnt from the third project. Another fully RESTful app (Who would have thought), this one was particularly fun to make, as Rails made about relationships between categories much easier to implement than Mongo IMO.

You can comment, upvote and downvote comments, post pictures with them, send people hatemail and nudes you know the usual (don't though, they go to my AWS account and I will leak them). The twist is that topics cannot be created by users, as it is a command that takes the top 5 Twitter trends located in USA live using the Twitter API, then creates topics from those.

Extremely fun to make and felt very simple due to Rails' shortcuts. I am definitely a fan of DHH.
